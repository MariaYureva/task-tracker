# Task Tracker API

API модуля трекера рабочих задач для медицинской информационной системы (МИС).
Ruby on Rails (API-only) + PostgreSQL.

Ключевая идея: **вхождения повторяющихся задач виртуальны**. В базе хранится
только правило серии; отдельная строка появляется лишь для того дня, который
«оторвался» от правила (сменили статус, перенесли, отменили или переопределили
текст). Это напрямую решает «проблему бесконечности»: бесконечная ежедневная
задача не порождает ни одной строки, пока её конкретный день не тронут руками.

## Стек

- Ruby 3.4, Rails 7.1 (API-only)
- PostgreSQL 16
- RSpec, FactoryBot, shoulda-matchers
- rswag (OpenAPI / Swagger UI)

## Запуск через Docker (рекомендуется)

```bash
docker compose up --build
```

Поднимется PostgreSQL и приложение. Энтрипоинт дожидается БД и идемпотентно
выполняет `db:prepare` (создание, миграции, сиды). API будет на
`http://localhost:3000`, Swagger UI — на `http://localhost:3000/api-docs`.

## Локальный запуск

```bash
bundle install
bin/rails db:prepare   # create + migrate + seed
bin/rails server
```

Конфигурация БД читается из переменных окружения
(`DATABASE_HOST`, `DATABASE_PORT`, `DATABASE_USERNAME`, `DATABASE_PASSWORD`,
`DATABASE_NAME`), значения по умолчанию — локальный postgres/postgres.

## Тесты

```bash
bundle exec rspec
```

## Идентификация и авторизация

Полноценная аутентификация (логин/токены) **намеренно вне рамок** тестового
задания. Текущий пользователь определяется по заголовку запроса `X-User-Id`.
При этом авторизация присутствует: пользователь видит и меняет только свои
задачи. Сиды создают двух демо-пользователей — их id можно получить через
`GET /api/v1/users` (этот эндпоинт не требует заголовка).
